# Pressure test of the latest findings

> **`DESIGN RECORD`** · shipped; kept for rationale · _reviewed 2026-06-13_

We attacked the three central claims — (1) the artifact is unfakeable, (2) the
diff catches what reviewers miss, (3) it reports only newly-introduced
violations — by trying to break them with adversarial inputs, not confirm them.
Two claims cracked and forced corrections; the corrected versions are stronger
and more honest.

---

## Attack 1 — re-signed forgery (digest alone is not tamper-proof)

**Attack:** the agent edits the artifact body (flip BLOCK→CLEAR, drop
violations) and then *recomputes the digest over the lie* using the same digest
function it has access to.

**Result: the forgery is self-consistent** — body hashes to its own digest — but
**caught** by the *code-correspondence* check, because the verifier recomputes
the artifact from the real base/branch graphs and gets a different digest.

**Finding (correction to the doc's framing):** the digest is **not a
cryptographic signature**, and "unfakeable digest" oversold it. The security
does **not** live in the digest — it lives in the **verifier re-deriving the
artifact from the source graphs**. The digest only catches accidental edits and
staleness. State it precisely: *the artifact is unforgeable only because a
trusted party recomputes it from the code; the number itself proves nothing.*

---

## Attack 2 — agent-controlled graph (the real trust anchor)

**Attack:** the guarantee depends on `base_graph` and `branch_graph`. If the
**agent** generates `branch_graph` (by running flowmap itself), it submits a
graph that simply omits the offending edge. The graph is internally valid; it
just doesn't match the code.

**Result: ATTACK SUCCEEDS.** The honest change renders BLOCK; the agent-supplied
doctored graph renders CLEAR, and nothing in groundwork can tell the difference —
the digest faithfully signs a graph that lies about the code.

**Finding (the load-bearing condition, previously unstated):** the entire
guarantee rests on one assumption — **the graph must be generated by trusted CI
from the actual branch code, never accepted from the agent.** This is *the*
security boundary. groundwork's verdicts are conditional on a trusted graph;
flowmap execution must sit inside the CI trust boundary, with the agent able to
*read* graphs but never *supply* the one it's judged against. Without that, every
downstream guarantee is void.

---

## Attack 3 — false negatives on the diff (known boundary, reconfirmed)

**Attack:** find dangerous changes that produce *zero* graph delta.

**Result, three classes ship with no structural signal:**
- **Body-only logic** (clamp `10`→`100`): zero delta. The literal is inside the body.
- **Swapped arguments** at an existing call (`(topicA, topicB)`→`(topicB, topicA)`): same `from→to` edge, empty delta. Argument order isn't in the edge.
- **New call to an already-called target:** the diff is *set-based*, so an edge that already exists elsewhere produces no "added edge" even if this site is new.

**Finding:** consistent with the stated boundary (structure, not logic), but the
set-based-diff blind spot (third bullet) is sharper than previously documented —
the diff sees *new edge types*, not *new call sites*. A function newly calling
something it already called from another site is invisible.

---

## Attack 4 — the dangerous framing (corrected in code)

**Attack:** how does a body-only MR present to a hurried reviewer?

**Result before fix: it rendered `CLEAR`.** A body-only change — *exactly where
logic bugs live* — got the same green verdict as a verified-safe structural
change. A reviewer trusting the green would **skip the logic review the change
most needs.** The tool would be actively harmful in the highest-risk case.

**Correction (shipped):** the verdict is now **three-valued**:
- `BLOCK` — a structural invariant or contract broke.
- `STRUCTURALLY-CLEAR` — shape preserved; explicitly *not* a logic/test sign-off.
- `NO-STRUCTURAL-SIGNAL` — body-only change; the graph **abstains**, and the
  report says so. It exits 0 (doesn't block CI) but is not a clean bill of health.

This makes the artifact's silence legible: a reviewer can no longer mistake
"the graph has nothing to say" for "the graph says it's fine." The abstention is
now a first-class signal.

## Net effect on the claims

| Claim | Verdict after pressure test |
|---|---|
| Unfakeable digest | **Corrected** — digest isn't the anchor; *recomputation from a trusted graph* is. |
| Trusted-graph assumption | **Newly surfaced** — the whole guarantee is conditional on CI-generated graphs; agent-supplied graphs forge a pass trivially. |
| Catches what reviewers miss | **Holds, with sharper limits** — set-based diff misses new call sites to already-called targets; all body-level/arg-level changes are invisible by design. |
| Reports only new violations | **Holds.** |
| Green = safe | **Corrected** — split into structurally-clear vs no-signal so green can't be misread as logic-reviewed. |

The honest one-line summary: **the tool's structural guarantees are real but
conditional, its silence is now explicit rather than disguised as approval, and
its single point of failure is the integrity of the graph it's handed — which
must come from trusted CI, never from the agent under review.**
